// python tooling

Security Scripts

Five purpose-built Python security tools covering reconnaissance, packet analysis, vulnerability triage, log hunting, and network baselining. All use Python 3.8+ stdlib where possible — minimal dependencies by design.

Python 3.8+ Stdlib-first No pip bloat scapy optional for PCAP tools
TCP Port Scanner
port_scanner.py
Reconnaissance Threading

Multithreaded TCP port scanner supporting single IPs, hostnames, and full CIDR ranges. Uses Python's socket and concurrent.futures — no nmap required. Outputs open ports with service name guesses and exports to CSV for reporting.

Key capabilities

CIDR range expansion 100-thread concurrency Service name mapping CSV export Configurable timeout
# Scan a host across top ports python port_scanner.py --target 192.168.1.0/24 --ports 22,80,443,3389,8080 # Full port scan with CSV output python port_scanner.py --target 10.0.0.1 --ports 1-65535 --output scan.csv ## Output: [OPEN] 192.168.1.1:22 (SSH) [OPEN] 192.168.1.1:80 (HTTP) [OPEN] 192.168.1.1:443 (HTTPS) Scan complete — 3 open port(s) found
PCAP Analyzer
pcap_analyzer.py
Threat Detection Wireshark Companion

Parses .pcap / .pcapng files with Scapy to produce a threat-intelligence-style summary: protocol breakdown, top talkers, suspicious port flags (Metasploit, Tor, IRC), cleartext credential indicators in FTP/Telnet/HTTP, and a full DNS query log. Exports each category to CSV.

Key capabilities

Protocol breakdown Top talker ranking Suspicious port flagging Cleartext credential detection DNS query extraction CSV report export
python pcap_analyzer.py --file capture.pcap --output ./report ## Sample output: PROTOCOL BREAKDOWN TCP 3412 ████████████████████████ UDP 891 ██████ ICMP 234 █ ⚠ SUSPICIOUS CONNECTIONS (2 found) [!] 192.168.1.105 → 10.0.0.3:4444 — Metasploit default [!] 192.168.1.105 → 10.0.0.9:6667 — IRC (C2 channel) ⚠ CLEARTEXT CREDENTIAL INDICATORS [!] 192.168.1.20 → 10.0.0.5:21 | keyword: 'PASS '
Nessus Report Parser
vuln_report_parser.py
Nessus / ACAS SAR Generation

Parses Nessus / ACAS CSV exports and auto-generates a prioritized remediation report. Ranks hosts by cumulative CVSS risk score, deduplicates findings, and outputs a Markdown SAR draft — cutting manual report-writing time significantly. Built directly from DoD compliance work at EHS Technologies.

Key capabilities

CVSS v2/v3 scoring Host risk ranking Severity breakdown Remediation queue Markdown SAR export
python vuln_report_parser.py --input nessus_export.csv --output report.md ## Terminal summary: Total findings: 33 SEVERITY BREAKDOWN Critical 3 ███ High 7 ███████ Medium 14 ██████████████ Low 9 █████████ TOP HOST BY RISK SCORE 1. 192.168.1.10 score=28.4 findings=6
Log Threat Hunter
log_threat_hunter.py
Brute Force Lateral Movement

Hunts security log files (Linux auth.log, /var/log/secure) for IoCs: SSH brute-force patterns, failed su attempts, suspicious sudo commands, new user creation, and successful authentications from IPs that previously failed — a strong lateral movement signal.

Detection signatures

SSH brute-force (≥5 fails) Invalid user attempts New account creation Password changes Post-failure success auth JSON findings export
python log_threat_hunter.py --log /var/log/auth.log --output findings.json ⚠ BRUTE-FORCE DETECTION [!] 185.220.101.47 247 failed SSH attempts [!] 194.165.16.22 89 failed SSH attempts ⚠ POSSIBLE LATERAL MOVEMENT [!] Line 8821: Success from IP 185.220.101.47 that also had failures → session opened for user deploy by (uid=0)
Network Baseline Comparator
network_baseline.py
Anomaly Detection Before/After

Builds a JSON network snapshot from a reference PCAP, then diffs a second capture against it — surfacing new hosts, newly opened ports, protocol shifts, and traffic volume spikes. Includes a demo mode that runs with synthetic data so you can test it without a real capture file.

Three modes

baseline — save snapshot compare — diff captures demo — synthetic data
# Build baseline python network_baseline.py --mode baseline --pcap week1.pcap --save snap.json # Compare new capture python network_baseline.py --mode compare --pcap current.pcap --baseline snap.json 🔴 NEW HOSTS DETECTED (1) [NEW] 192.168.1.99 ports=[4444, 8888] 🟠 NEW OPEN PORTS ON EXISTING HOSTS (1) [!] 192.168.1.10 new ports: [4444] 🟡 TRAFFIC VOLUME SPIKES (1) [!] 192.168.1.10 7.5x increase (120,000 → 900,000 bytes)
